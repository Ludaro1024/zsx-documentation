## Jak prawidłowo zainstalować zasób Multicharacter

Aby upewnić się, że zasób został poprawnie zainstalowany i działa, postępuj zgodnie z poniższymi krokami:

import { Code, Steps, Tabs, TabItem } from '../../@astrojs/starlight/components'

<Tabs>
  <TabItem label="ESX">
    <Steps>
      1. **Skonfiguruj ESX**  
         Jeśli korzystasz z ESX, upewnij się, że poniższe ustawienie jest włączone w pliku `es_extended/config.lua` lub `es_extended/shared/main.lua`:  
         <Code code='
         Config.Multichar = true (GetResourceState("esx_multicharacter") ~= "missing")
         'lang="lua" label="es_extended/config.lua" del='GetResourceState("esx_multicharacter") ~= "missing"'> 
         </Code>

         Usuń czerwoną część i zamień ją na true.

      2. **Pobierz zasób**  
         Pobierz zasób Multicharacter z [Keymaster site](https://keymaster.fivem.net/).

      3. **Dodaj zasób do serwera**  
         Przenieś pobrany zasób do głównego katalogu `/resources`.

      4. **Zaktualizuj plik konfiguracyjny**  
         Otwórz plik `server.cfg` (lub użyj txAdmin > CFG Editor) i wykonaj następujące kroki:  
         Dodaj poniższe linie, aby zatrzymać istniejące zasoby multicharacter, jeśli ich jeszcze nie usunąłeś:  
           ```plaintext
           stop esx_multicharacter  # Domyślny ESX multicharacter
           ```
         Dodaj poniższą linię, aby upewnić się, że nowy zasób Multicharacter uruchamia się:  
           ```plaintext
           ensure ZSX_Multicharacter
           ```
           Ta linia powinna być umieszczona zaraz po `ensure` twojego frameworka.

      5. **Ustaw liczbę postaci**  
         Skonfiguruj maksymalną liczbę postaci w pliku `/shared/config.lua`:  
         ```lua
         Config.Characters.Free = 1  -- Zamień "1" na pożądaną liczbę darmowych postaci
         ```

      6. **Uruchom serwer**  
         Uruchom serwer i ciesz się zasobem!
    </Steps>
  </TabItem>
